# Product

## Register

brand

## Users

Clientes potenciales de proyectos freelance, empleadores y colaboradores del ámbito tech. Los visita desde una recomendación, un perfil de GitHub o LinkedIn. Tienen entre 5 y 30 segundos para decidir si siguen leyendo. Quieren ver capacidad técnica y criterio de diseño al mismo tiempo.

## Product Purpose

Portfolio personal de Bautista Kahr: demostrar que puede construir sitios web reales, bien hechos, con criterio visual propio. El objetivo es conseguir proyectos freelance y oportunidades laborales. El éxito es que alguien llegue y mande un mensaje de contacto.

## Brand Personality

Creativo, bold, enérgico. El portfolio en sí es una demostración de lo que Bautista puede hacer: si el sitio es espectacular, el argumento de venta se hace solo. El tono es confiado sin ser arrogante, técnico sin ser frío.

## Anti-references

- Portfolios genéricos estilo Awwwards: glassmorphism decorativo, gradient text, hero con métricas tipo "500+ clientes".
- El template de React + Tailwind de GitHub: timeline de experiencia, íconos de stack en fila, todo igual.
- Feel corporativo o LinkedIn: foto formal, barras de progreso de habilidades, lenguaje de CV.
- Aspecto de "generado por IA": cualquier patrón que parezca el primer resultado de un generador. Eyebrows en todas las secciones, paleta beige/crema, hero-metric template.

## Design Principles

1. **El sitio es el portfolio.** No decir "soy bueno en diseño" — demostrarlo con cada píxel del propio sitio.
2. **Movimiento con propósito.** Cada animación tiene que agregar algo o no estar. El craft visual es el argumento.
3. **Oscuro y denso, no minimalista.** La personalidad vive en la atmósfera: grain, blobs, partículas, glow. No limpiar por limpiar.
4. **Confianza sin barroquismo.** Bold en la tipografía y el color, disciplinado en el layout. El exceso visual mata la credibilidad.
5. **Consistencia sistémica.** Todos los tokens, todos los componentes hablan el mismo idioma. El sistema se nota cuando falta.

## Accessibility & Inclusion

WCAG AA: contraste de texto ≥ 4.5:1, navegación por teclado funcional, reduced-motion respetado. Screen readers no son prioridad máxima pero el HTML semántico debe ser correcto.
